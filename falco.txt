## üß© Option 1 ‚Äî Manual Command (Trivy v0.67.2, Offline Mode)

Since your version (v0.67.2) doesn‚Äôt support the `--namespace` flag,
you can use this **two-step command** approach:

```bash
# Replace <namespace> with your actual namespace (e.g. springboot-bau)
kubectl get all -n <namespace> -o yaml > /tmp/<namespace>_all.yaml

trivy k8s --input /tmp/<namespace>_all.yaml \
  --report summary \
  --format html \
  --cache-dir /root/.cache/trivy \
  --skip-db-update \
  --offline-scan \
  --output /var/log/trivy/trivy_<namespace>_report_$(date +"%Y-%m-%d_%H-%M").html
```

‚úÖ Example for your case:

```bash

trivy k8s --input /tmp/springboot-bau_all.yaml \
  --report summary \
  --format html \
  --cache-dir /root/.cache/trivy \
  --skip-db-update \
  --offline-scan \
  --output /var/log/trivy/trivy_springboot-bau_report_$(date +"%Y-%m-%d_%H-%M").html
```

Then open:

```
/var/log/trivy/trivy_springboot-bau_report_<timestamp>.html
```

Or view it in browser:

```
http://<master-node-ip>:8080/trivy/trivy_springboot-bau_report_<timestamp>.html
```

## üß© Option 2 ‚Äî Quick Text Output (for console view)

If you just want a terminal summary instead of HTML:

```bash
kubectl get all -n <namespace> -o yaml > /tmp/<namespace>_all.yaml

trivy k8s --input /tmp/<namespace>_all.yaml \
  --report summary \
  --format table \
  --cache-dir /root/.cache/trivy \
  --skip-db-update \
  --offline-scan
```

Example:

```bash
trivy k8s --input /tmp/springboot-bau_all.yaml --report summary --format table --cache-dir /root/.cache/trivy --skip-db-update --offline-scan
```

This gives you a clear console table output with CVEs, severities, and packages ‚Äî no HTML file.
